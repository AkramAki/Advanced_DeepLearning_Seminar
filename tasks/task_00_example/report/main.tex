\section{Goal}

This task demonstrates the full workflow of the repository:
code $\rightarrow$ plots $\rightarrow$ report.

The goal is to show how a weekly task can be organized, how figures are generated
with Python, how they are included in \LaTeX{}, and where the final PDF is saved.

\section{Bibliography}

References are stored globally in

\begin{center}
  \texttt{latex/references.bib}
\end{center}

They are loaded through the shared header file. This means every task can use the
same bibliography database.

For example, a source can be cited with

\begin{center}
  \texttt{\textbackslash cite\{example-key\}}
\end{center}

or, depending on the context,

\begin{center}
  \texttt{\textbackslash autocite\{example-key\}}
\end{center}

The bibliography is printed at the end of the report with

\begin{center}
  \texttt{\textbackslash printbibliography}
\end{center}

\section{Build Output}

The task is built using \texttt{make}. Running

\begin{center}
  \texttt{make}
\end{center}

inside the task folder first generates the figures and then compiles the report.

Temporary build files are written to

\begin{center}
  \texttt{build/}
\end{center}

The final PDF is copied to

\begin{center}
  \texttt{report/main.pdf}
\end{center}

This is the file that should be opened or submitted.

\section{How it Works}

\begin{itemize}
  \item Python scripts are stored in \texttt{code/}
  \item Plots are generated by \texttt{code/make\_plots.py}
  \item Figures are saved in \texttt{../build/}
  \item The report is written in \texttt{report/main.tex}
  \item External PDF figures are included with \texttt{\textbackslash includegraphics}
  \item Bibliography entries are stored in \texttt{latex/references.bib}
  \item Temporary files are collected in \texttt{build/}
  \item The final report is saved as \texttt{report/main.pdf}
\end{itemize}

\section{Conclusion}

This setup allows reproducible experiments and clean reports. Each task stays
self-contained, while shared settings such as the \LaTeX{} header, bibliography,
and matplotlib configuration are reused across the repository.
